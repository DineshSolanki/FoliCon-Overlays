# FoliCon-Overlays

Community overlay repository for [FoliCon](https://github.com/DineshSolanki/FoliCon).

## What are overlays?

Overlays define custom poster icon styles — mockup frames, rating badges, title text, and layer compositions. FoliCon's Overlay Store lets you browse and install overlays from this repository with one click.

## Creating an Overlay

**📖 Read the full guide: [Creating Overlays](CREATING-OVERLAYS.md)** — everything you need to design, build, and submit an overlay.

Quick version:

1. Fork this repository
2. Copy an existing overlay folder (e.g. `overlays/example-dvd-case/`) as a starting point
3. Edit `overlay.json` — see the [overlay schema](schemas/overlay-schema.json) for all fields
4. Edit `manifest.json` — fill in metadata, list all assets, compute SHA256 hashes
5. Create a `preview.png` (256x256 PNG) showing how the overlay looks
6. Submit a Pull Request

## Validation Limits

- **Image size**: max 2MB per image
- **Package size**: max 5MB total per overlay
- **Format**: PNG only
- **Schema**: must pass `overlay-schema.json` validation

## Manifest

Each overlay folder must contain a `manifest.json` with:
- `id` — unique lowercase identifier with hyphens (matches folder name)
- `displayName`, `author`, `description`
- `overlayVersion` — semver string
- `assets` — list of all files in the folder
- `sha256` — per-file SHA256 hashes (hex-encoded, 64 chars)
- `previewImage` — filename of the preview image

See [manifest-schema.json](schemas/manifest-schema.json) for the full schema.

## Auto-generated Catalog

The `catalog.json` at the repository root is auto-generated by GitHub Actions on every push to `main` that changes files under `overlays/`. **Do not edit it manually.**
